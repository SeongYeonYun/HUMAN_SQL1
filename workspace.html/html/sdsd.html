A. 데이터 모델링 Essentials

1) 핵심 개념

  • 목적: 현실을 추상화·단순화·명확화하여 재사용 가능한 구조로 표준화
  • 관점: 데이터(What), 프로세스(How), 상관(Interaction)
  • 3단계: 개념→논리→물리 (정규화·키·무결성 → 저장구조·인덱스)

2) 4대 구성요소

  • 엔터티: 관리 대상의 집합, 식별자·속성 보유
  • 속성: 더 이상 쪼갤 수 없는 의미 단위(기본/설계/파생)
  • 관계: 존재/행위 기반, 1:1·1:N·N:M, 필수/선택
  • 식별자: 주/보조·내부/외부·단일/복합·본질/인조

3) 식별자 관계 vs 비식별자 관계

  • 식별자 관계: 부모 PK가 자식 PK로 상속 → 1:1 또는 1:N
  • 비식별자 관계: 부모 키를 자식의 일반 FK로만 보유(Null 허용/수명주기 분리 등)
  • PK 속성 남발(연쇄 상속) 시 조인 키 과다 → SQL 복잡·성능 저하

4) 좋은 모델의 조건

  • 완전성·일관성·중복 배제·명확한 규칙·단순성·확장성

B. SQL 기초·중급 문법 총정리

1) 집합과 조인(Join)

  • INNER / OUTER JOIN (LEFT/RIGHT/FULL), CROSS JOIN — 카티전 곱 p. ~150 전후
  • JOIN 조건 누락 시 데이터 폭증(카디널리티 M×N)

2) 비교·범위·리스트

  • BETWEEN 경계 포함 유의(날짜·시분초 포함범위) p. ~210
  • IN(list) vs 다중 OR — 옵티마이저 변환 특성
  • 부정: NOT IN 시 NULL 주의(삼진 논리)

3) NULL 정확히 다루기

  • NVL/ISNULL/COALESCE 차이, NULLIF 용도 p. ~231
  • 집계/산술 시 NULL 전파, 외부조인과의 상호작용

4) 행 제한

  • ROWNUM / TOP / FETCH FIRST … ONLY / LIMIT p. ~211

C. SQL 고급(윈도우·서브쿼리·CTE)

1) 윈도우 함수

  • LAST_VALUE(... IGNORE NULLS) 등 누적/누락 보정 p. ~311
  • 파티션·정렬·프레임(ROWS/RANGE) 선택이 결과/성능 좌우

2) CTE(공통 테이블 식, WITH)

  • 가독성↑, 재사용↑, 재귀 쿼리·단계적 집계에 유리 p. ~311
  • DBMS에 따라 머티리얼라이즈/인라인 최적화 차이

3) 서브쿼리 활용

  • EXISTS vs IN, 세미/안티 세미 조인 전략
  • 스칼라 서브쿼리 — 선택적으로 인덱스 스캔 유도

D. 옵티마이저 & 실행계획 이해

1) CBO(비용기반) 기본

  • 통계(카디널리티·히스토그램·데이터 분포) 기반 경로 선택 p. ~251
  • 블록 I/O·CPU 비용 모델, 조인 순서/방식 결정

2) 실행계획 읽기

  • Access vs Filter, Predicate 정보, 예상/실측 Rows, 비용(=비교지표) p. ~151
  • 힌트는 보장이 아니라 유도 — 과용 금물
통계 갱신 주기 미흡·왜도(Skew) 미반영·파라미터 의존성은 잘못된 경로를 유발.

E. 인덱스·조인·소트 튜닝

1) 인덱스 구조 & 설계

  • B*-Tree 기본 원리, 선행 컬럼 선택도 중심 설계 p. ~271
  • 범위조건은 폭이 넓음 → IN-List 변환이 유리한 케이스 존재 p. ~291
  • 컬럼 순서: = 조건 → 범위 순으로 앞쪽 배치(조합 인덱스)

2) 조인 전략

  • NL/해시/소트머지 — 데이터량·선택도·메모리·조인 조건에 따라 선택
  • 드라이빙/드리븐 테이블, 조인 칼럼 인덱싱

3) 소트(정렬) 최소화

  • ORDER BY/윈도우/중복제거 DISTINCT는 소트 코스트↑
  • 가능하면 인덱스 정렬 재사용, 불가 시 소트 영역 최적화 p. ~331
“select nvl(max(주문번호),0)+1”로 일련번호 생성은 경합·갭·동시성 문제 → 시퀀스/아이덴티티 사용 p. ~331

F. 스키마/DDL/제약조건·무결성

1) DDL 변경

  • 칼럼 타입/크기 변경 시 영향도(인덱스·뷰·프로시저) 분석 p. ~171
  • 제약조건 추가/수정/삭제: ADD/DROP CONSTRAINT p. ~171

2) 무결성

  • PK/UK/FK/Check/Not Null — 논리 규칙을 물리로 보강
  • FK 인덱스는 조인·삭제 지연 방지에 효과(카스케이드 주의)

G. 성능 데이터 모델링 & 분산

1) 반정규화(De-Norm)

  • 목표: 조인 경로 단축·I/O 감소·복잡 SQL 단순화
  • 방법: 테이블/컬럼/관계 중복·분리·통합(무결성 관리 프로세스 필수)

2) 슈퍼/서브타입 물리 변환

  • 트랜잭션 패턴 기준으로 개별 vs 통합 테이블 결정

3) 대용량 전략

  • 파티셔닝(수평/수직), 이력 테이블, 집계 테이블, 클러스터링
  • PK/FK 순서 — 인덱스 활용 극대화(=조건을 앞)

4) 분산 DB 개요

  • 투명성(분할/위치/중복/장애/병행), 위치/분할/복제/요약 분산
  • 부분복제(지사→본사), 광역복제(본사→지사), 분석/통합 요약

부록 — 실무 체크리스트
