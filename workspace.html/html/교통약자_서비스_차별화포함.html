주제 : 교통약자 서비스

■ 서비스 개요

  • 고령자 및 교통약자를 위한 예약형 이동 서비스
  • 자원봉사 기반 Ride Credit 시스템 도입
  • 실시간 위치 추적, 사용자 맞춤형 UI, 관리자 통계 포함

■ 작동 원리

  1. 사용자는 웹/모바일 화면에서 회원가입 및 로그인
  2. 이동이 필요한 경로를 입력하여 예약을 신청
  3. 백엔드(Spring Boot)는 해당 요청을 REST API로 수신
  4. 자원봉사자 운전자와의 매칭 수행
  5. 이동 완료 후 자원봉사자에게 Ride Credit 적립, 관리자는 통계 확인
  6. 위치 정보는 실시간 지도 API와 연동, 데이터는 Oracle DB에 저장

■ 사용자 구성

  • 일반 사용자 (고령자, 교통약자)
  • 운전자 (자원봉사자 또는 계약 기사)
  • 관리자 (예약 승인, 통계 확인)

■ 기술 스택

  • 프론트엔드: HTML, CSS, JavaScript
  • 백엔드: Java(Spring Boot)
    → Python(Django/Flask)도 가능하지만, 안정성과 팀 호환성을 고려해 Spring 사용
  • DB: Oracle
    → MySQL도 가능하나 학습 기반은 Oracle
  • API: RESTful JSON
  • 지도 API: KakaoMap 또는 Google Maps
  • 인증: JWT, OAuth
  • 배포: Docker, Tomcat, Nginx

■ 기술 선택 이유

  • Spring Boot: 구조화와 보안성, 확장성 고려
  • Oracle: 중요한 데이터 처리에 안정적
  • JavaScript: 다양한 기기 호환
  • 지도 API: 실시간 위치 시각화, KakaoMap 추천

■ 주요 기능

  • 회원가입 및 로그인
  • 예약 신청 및 배정
  • Ride Credit 적립
  • 실시간 이동 추적
  • 관리자 통계 기능

■ 추천 아키텍처

  • [Client] - HTML, JS, CSS
  • [Spring Boot REST API]
  • [JPA/Hibernate]

    • Hibernate = Hibernate에는 JDBC와 JPA가 있다.

    JPA = 단순한 예약, 사용자, 크레딧 등 CRUD 중심이라 JPA만으로 충분하다.

  • [Oracle]
  • 지도 API, JWT 인증

    • JWT = 웹/모바일 인증 시스템에서 사용되는 토큰 기반 인증 방식.
