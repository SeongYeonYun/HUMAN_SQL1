병원 예약·이동·돌봄 통합 서비스 해커톤 기획안

1. 핵심 문제 정의

  • 고령자/IT 비숙련자의 병원 예약 접근성 부족
  • 교통약자 대상 실시간 위치 추적 및 ETA(예상 도착시간) 미지원
  • 동행자 부재 및 낯선 인력에 대한 보호자의 불안
  • 진료 이후 돌봄 연계 및 정서지원 단절

2. 제안 서비스 개요 (가칭: 헬프케어)

“병원까지의 모든 여정을 연결하는 고령자 맞춤형 통합 플랫폼”

  • 병원 예약 → 차량 매칭 → 동행 → 진료 → 귀가 → 돌봄 연계까지 원스톱 제공
  • Elder Village 커뮤니티 철학 기반: 이웃 중심 자율 돌봄 문화 포함

3. 주요 기능

  • 음성 및 챗봇 기반 병원 예약 / 보호자 대리 예약
  • 실시간 차량 위치 지도 표시 + ETA 안내
  • 이용자 상태(Level 1~3)에 따른 인력 자동 매칭
  • 귀가 후 복약 알림, 안부 확인, 영상통화 기능

4. 사용자 레벨 기반 인력 배치

  • Level 1: 경증 – 자원봉사자 매칭
  • Level 2: 중간 – 요양보호사 매칭
  • Level 3: 중증 – 간호인력/의료 연계

5. 신뢰 기반 설계: 프로필 & 평가 시스템

  • 모든 봉사자/지원자는 프로필 등록 필수 (자격증, 소속, 활동 이력 포함)
  • 이용자·보호자가 남긴 후기 및 평점을 누구나 열람 가능
  • 신뢰할 수 있는 인력은 “즐겨찾기”로 등록하여 반복 매칭 가능
  • 인증된 인력만 Level 2 이상 배정

6. 실시간 보호자 연계 기능

  • 보호자는 부모님의 현재 위치 및 차량 도착 시간 확인 가능
  • 진료 접수/시작/종료 단계마다 알림 및 메시지 전송
  • 보호자–지원자 간 사전 통화 및 채팅 기능
  • 응급상황 발생 시 보호자 및 119 자동 통보 시스템 포함

7. 보상 및 상생 구조

  • 자원봉사자: 지역 포인트 / 돌봄 크레딧 제공 (식당, 약국, 카페 사용 가능)
  • 요양보호사, 간호 인력: 유료 정식 계약 기반 활동
  • 돌봄 제공자는 “내 부모도 언젠간 도움받는다”는 상호순환 시스템의 일부

8. 기술 구성 (Tech Stack)

  • 프론트엔드: React.js 또는 Flutter
  • 백엔드: Spring Boot(Java) 또는 Flask(Python)
  • DB: MySQL 또는 Oracle
  • API: 병원 예약 API, Kakao/Tmap 위치 공유, 보호자 알림, 인증 시스템

9. 차별화 포인트 요약

  • ✅ 실시간 차량 위치 추적 + ETA 제공 (기존 교통약자 서비스 미제공)
  • ✅ 봉사자·요양보호사 프로필, 평점, 후기 열람 가능
  • ✅ 반복 매칭으로 신뢰 관계 지속
  • ✅ Elder Village 철학 반영: 지역 공동체 기반 자율적 돌봄 생태계 형성
  • ✅ 보호자 실시간 위치 확인 + 알림 + 채팅 기능
